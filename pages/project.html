The Project

Introduction

The work presented in this website is part of a project called giCASES whose objective is to create a University-Enterprise Alliance for a Spatially Enabled Society.

This Knowledge Alliance is co-funded by the Erasmus+ programme of the European Union that involves various European universities â€“ and Politecnico di Milano among them - with the purpose of reducing the gap between the knowledge students obtain while studying and the knowledge required once they start working. The idea is that this goal can be achieved through a collaboration with relevant companies presenting real life cases for the students to solve.

For this project, the students of the Geographic Information System course of professor Brovelli teamed up with Epsilon Italia to develop the case study CS5 - Harmonizing data flows in Energy Saving EU policies.

INSPIRE

The cornerstone of the project is the INSPIRE Directive with a focus on how states of the European Union are required to transform their data into an INSPIRE compliant format.

This will lead to the creation of a common spatial data infrastructure, enabling the sharing of spatial information between organisations and countries all over Europe and assisting them in policy-making across boundaries.

The work

The main goal of the project was to harmonize data flows related to energy saving policies for buildings located in the centre of Sondrio, one of the provincial capitals of Lombardy region in Italy.

In particular, the two objectives of the work were:

  • perform a transformation of the CENED (Certificazione ENergetica degli EDifici) dataset according to the extended INSPIRE Building 2D core data model;
  • show the CENED dataset of Sondrio together with additional ancillary information using a WebGIS.

The data pre-processing

Before starting the actual work, an understanding of the main goal of the INSPIRE Directive and the study of both the source data model and sets given from CENED and the target data model and schema from INSPIRE were required.

CENED data are downloadable in csv format for the whole region of Lombardy. With the csv in hand, we imported the data in QGIS to visualize and correctly georeference them.

For the following operations we needed about 100 point features, so we cut the dataset using a rectangular bounding box centered on our main city (Sondrio).
We obtained a bit more than 100 points and we saved them as shape file, downloadable here.

The transformation

Once this knowledge was acquired, the first step in harmonizing the data was to link the simpler source data model to the more complex target data model, matching attributes and values.

Mapping table

In order to visualize the attributes needed from the target data model and the corresponding attributes in the source data model, a mapping table was created starting from the EULF - Energy Pilot - UC1 uml model.

The process of compiling the map was based on two steps.

Firstly we were asked to transform the entities described in the uml to a table adding the relevant attributes (e.g. the voidability, the cardinality, the type of the value). Particular attention had to be given to the type of the attributes, which can be a primitive (e.g. integer, string, etc), a codelist, that is a fixed list of possible values, or another attribute table.

The second step was to associate all the attributes of the source model that had a corresponding one in the target.

Hale Studio

Based on the mapping table, we were able to perform the transformation using Hale Studio, an open source data transformation software.

As a preliminary operation we imported the source and the target schema as well as the source data in order to have a live validation of the work. We also imported all the codelists from the INSPIRE website.

Once we had all the needed data in hand we connected the two schemas with a Retype function.
After that, using various relations, we joined the attributes of the two tables.

  • We used Rename when the source and the target attributes were semantically equal.
  • The Classification function was instead used when we needed to move from a multiple boolean attribute in the source to a unique attribute in the target. This type of transformation is based on a codelist, and the Classification allowed us to link a true value in the source to the corresponding value in the codelist.
  • We used Assign to put a value in the target schema that was not present in the source but was known for all the table. A slightly different version of the function, Assign(Bound) was also used in order to deal with the attributes containing some null values.
  • Formatted string was used when we had to join multiple source attributes in one target attribute.
  • Finally, in order to assign a unique id to the sub-table we used Generate Unique Id or Generate Sequential Id.

Once all the attributes had been linked, we performed a validation of the final schema using both a project validator and an XML validator.
The validation highlighted no errors, but only a couple of warnings related to the fact that we added some custom values to a few codelists to better fit the source data.

The Hale project and the final .GML (Geography Markup Language) output can be downloaded here.

The Web GIS

The last part of the project consisted of creating a WebGIS to visualize the transformed data along with various basemaps and ancillary datasets related to the energy topic, all published as WMS or WFS using GeoServer.

Using OpenLayers we set up a WebGIS which shows a visualization of:

  • The original CENED dataset, consisting of a selection of 103 buildings in the centre of Sondrio, provided as WFS. This allows the user to get some information about the building by clicking on the corresponding point. The information showed are about the address of the building (field INDIRIZZO), the id of the certificate (field COD_APE), the energetic class and in some cases the proposed one (fields CLASSE_EN and RIQ_CLASSE).
  • The boundaries of the municipality and of the province of Sondrio in 2016 taken from the ISTAT (Istituto Italiano di Statistica Italian Institute of Statistics) Website.
  • The building footprint cut in the center area of Sondrio taken from the the Lombardy Region Geoportal Website named Database topographico edificato.
  • Finally, as ancillary layers one can find the dams, the hydraulic regulation works and the geothermal plants in the area of Sondrio. The choice had fallen upon those constructions because they either provide or have the potential to provide electric or heat energy for housing. This data are from the Lombardy Geoportal Region too.

The map is completed by the CENED dataset of Lecco and a choice of four different basemaps:

  • OpenStreetMap (OSM);
  • Bing Maps - Roads;
  • Bing Maps - Aerial with labels;
  • Stamen toner.
